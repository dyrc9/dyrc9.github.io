Embodied intelligence notes 7 - world models for embodied agents

前面 RL 笔记里写过 model-based RL 和 world model。放到具身智能里，这个话题会变得更具体一点。

机器人不是在一个干净模拟器里选动作。它看到的是部分观测，动作会进入物理世界，失败有成本。所以如果模型能在内部先想一想“做这个动作会怎样”，就很有价值。

但这里的 world model 也不能理解得太浪漫。它不一定要完整复刻世界，更现实的目标是：预测和任务相关的变化，帮助策略少犯一些贵的错。

model what

world model 到底要建模什么？

可能有几种层次：

  • pixel-level next frame
  • latent state transition
  • object pose change
  • contact outcome
  • task progress
  • failure probability
  • affordance

预测下一帧图像看起来最直观，但不一定最有用。机器人关心的可能不是每个像素，而是杯子有没有被抓住、抽屉有没有打开、物体会不会滑落。

planning in latent space

有了 dynamics model，就可以在内部 rollout：

这有点像 MPC。每次从当前观测出发，试几组候选动作，选一个预测结果最好的，然后只执行前几步，再重新观察。

对机器人来说，这个结构很自然：

  • 真实执行贵
  • 预测执行便宜
  • 环境会变化，所以要不断重新规划

但 latent planning 的难点也很明显：如果模型预测错了，planner 可能会利用模型漏洞，选出真实世界里不可行的动作。

这和 RL 里的 model exploitation 是同一个问题。模型不是环境，planner 太相信模型会出事。

这和 diffusion policy 也能接上。扩散模型可以采样多条 action chunk，world model 或 critic 在里面挑更稳的一条。

在这个意义上，world model 更像一个想象和筛选模块，而不是直接替代 policy。

counterfactual

world model 很有用的一点是做 counterfactual。

真实机器人不能随便试：

  • 如果从左边抓会怎样？
  • 如果先推一下再抓会怎样？
  • 如果夹爪闭合慢一点会怎样？
  • 如果失败后换一个角度能不能恢复？

这些问题如果全部在真实世界里试，成本很高。world model 可以先给一个粗略答案。

当然，这个答案不一定准。尤其是接触丰富的任务，预测会很难。但哪怕模型只能排除明显不好的动作，也有用。

temporal abstraction

长程任务里，预测每一帧会很累，也容易误差累积。

所以 world model 也可以建在 skill 层：

比如：

  • grasp cup 之后，杯子应该在夹爪里
  • open drawer 之后，抽屉状态应该变成 open
  • place object 之后，目标位置应该有物体

这种抽象预测更适合规划长任务。它不关心每个关节怎么动，只关心 skill 执行后的世界状态。

这和 hierarchical RL 的 option model 有点像。option 不只是一个动作宏，也可以有自己的 transition model。

uncertainty matters

world model 不能只输出一个确定预测。机器人里不知道就是不知道，这件事最好显式表示出来。

比如模型可以告诉 planner：

  • 这个动作结果很确定
  • 这个动作可能成功也可能失败
  • 这个状态训练数据里很少见
  • 这个接触结果我预测不了

uncertainty 可以用来做两件事：

  • 避免高风险动作
  • 指导下一批数据收集

如果模型在某类状态下总是不确定，那就是数据缺口。这里又接回 data flywheel：world model 的不确定性可以告诉我们应该去哪里补数据。

sim and real

world model 和 sim-to-real 之间也有关系。

传统 simulator 是一种手写 world model。学习到的 world model 则更像从数据里拟合出来的 simulator。

两者可以混用：

  • simulator 给大体物理结构
  • learned model 补偿真实误差
  • real data 校准视觉和接触

这和 residual learning 的思路很像。真实世界很难完全建模，那就承认误差存在，然后让数据去补。

小结

具身智能里的 world model 可以先记：

  • 不一定要预测完整图像，更重要的是预测任务相关变化
  • latent planning 可以减少真实试错，但会有 model exploitation
  • policy 可以提出动作，world model 负责想象后果和筛选
  • counterfactual 对高成本真实实验很有价值
  • skill-level world model 更适合长程任务
  • uncertainty 既能避险，也能指导数据收集
  • simulator 和 learned model 可以互补

如果说 policy 是“我现在想怎么动”，world model 更像“我先想一下这样动之后会发生什么”。在真实机器人里，这个“先想一下”往往能省掉很多昂贵的试错。
