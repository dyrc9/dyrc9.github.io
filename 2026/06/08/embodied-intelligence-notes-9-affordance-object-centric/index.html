Embodied intelligence notes 9 - affordance and object-centric representation

前面写 VLA、world model、memory 时一直绕不开一个问题：机器人到底应该怎么看世界？

普通视觉模型经常回答“这是什么”。但机器人还要回答“它能怎么被操作”。同样是一个杯子，对分类模型来说是 cup；对机器人来说，还要知道哪里能抓、杯口在哪里、放下时哪里稳定、会不会滑。

这就是 affordance 这条线。

object name is not enough

知道物体类别不等于知道怎么行动。

比如识别出“抽屉”之后，还需要知道：

  • 把手在哪里
  • 往哪个方向拉
  • 当前是开还是关
  • 拉动时会不会撞到别的东西
  • 需要多大力

识别出“杯子”之后，也还要知道：

  • 能不能从侧面抓
  • 杯子是不是透明
  • 有没有液体
  • 杯柄是否可见
  • 放到哪里不会倒

所以 perception 不能只把图像压成一个语义标签。它需要保留和动作有关的结构。

affordance

affordance 可以粗略理解成“某个对象或区域支持什么动作”。

比如：

  • handle affords pulling
  • flat surface affords placing
  • cup side affords grasping
  • button affords pressing
  • open space affords moving through

它不是纯视觉属性，也不是纯语言概念，而是物体、机器人身体和任务之间的关系。

同一个物体，对不同机器人 affordance 也可能不同。人手能捏住的薄片，夹爪未必能抓；移动机器人能通过的缝，机械臂未必能伸进去。

所以 affordance 本质上带着 embodiment。

object-centric representation

object-centric representation 的目标是把世界组织成对象和关系，而不是一整块像素。

可以想象成：

这种表示对长程任务很有用。因为任务通常也是围绕对象说的：

  • pick up the cup
  • open the drawer
  • put the spoon in the bowl

如果模型内部没有对象结构，就很难稳定跟踪“哪个杯子已经处理过”“抽屉现在是不是开了”。

当然，object-centric 不一定要手写成 scene graph。它也可以是模型学出来的 slot、keypoint、region token 或 latent object embedding。关键是表示里要有可追踪的对象单位。

keypoints and contact

机器人动作经常和关键点有关。

抓取不只是抓“杯子”，而是抓某个位置。开门不只是对“门”动作，而是对把手动作。插入任务也常常依赖孔、边缘、轴线这些几何结构。

所以很多系统会预测：

  • grasp point
  • handle point
  • contact point
  • object keypoints
  • target pose
  • insertion axis

这些比类别标签更接近控制。

视觉模型如果能输出稳定 keypoints，后面的 planner 或 controller 会轻松很多。反过来，如果视觉只输出一段模糊 embedding，低层策略就要自己从里面解出几何关系，难度会高一些。

relation to language

语言里经常包含对象和关系：

这里至少有：

  • target object: red cup
  • destination: plate
  • relation: on

VLA 要做 grounding：把语言里的对象和图像里的对象对上。

如果场景里有两个杯子，一个红色一个蓝色，模型必须知道“red cup”对应哪个视觉区域。之后 memory 还要继续跟踪这个对象，不然中途遮挡或移动后会丢目标。

所以语言 grounding、object-centric perception 和 memory 是连在一起的。

affordance and world model

world model 也需要 affordance。

如果模型知道“这个把手可以被拉”，它就不必从零预测所有像素变化。它可以在更抽象的层面预测：

这比直接预测下一帧图像更接近任务。

同样，failure prediction 也可以用 affordance：

  • 抓取点太靠边，可能 slip
  • 放置面太小，可能 unstable
  • 夹爪路径被遮挡，可能 collision
  • 物体太薄，可能 grasp failure

这些都是 action-conditioned perception。不是只看世界是什么，而是看某个动作在这个世界里会不会成立。

learning affordance

affordance 可以来自几种数据：

  • 人工标注的可交互区域
  • 机器人真实尝试的成功/失败
  • 人类视频里的接触和操作
  • 仿真中自动生成的交互结果
  • language supervision

其中最有价值的往往是真实交互数据。因为 affordance 最终要被动作验证：看起来能抓，不等于真的能抓。

这和 RL 的关系也很明显。RL 或 self-supervised interaction 可以让机器人主动试，记录哪些动作成功，哪些失败。久而久之，视觉表征会更贴近“能不能做成事”。

representation for control

最后还是要问：这个表示怎么进入控制？

几种方式：

  • policy 直接吃 object tokens
  • planner 在 scene graph 上规划
  • affordance map 生成候选动作
  • world model 预测 object state change
  • controller 追踪 keypoint 或 target pose

不同系统会选不同接口。端到端模型可以少写很多规则，但也更难 debug。显式对象和 affordance 表示可解释一些，但需要更好的检测、跟踪和标定。

这里没有绝对答案。我的感觉是，越接近真实机器人部署，越需要一些可检查的中间表示。至少要知道失败是 perception 错了、affordance 判断错了，还是 controller 执行错了。

小结

affordance 和 object-centric representation 可以先记：

  • 物体类别不等于可操作信息
  • affordance 是对象、动作、任务和机器人身体之间的关系
  • object-centric representation 有助于长程任务和 memory
  • keypoints、contact points、target pose 比类别标签更接近控制
  • language grounding 需要把语言对象对到视觉对象
  • world model 可以在 affordance 层预测状态变化和失败风险
  • 真实交互数据对学习 affordance 很重要

具身智能里的 perception 最终不是为了“看懂一张图”，而是为了让动作更可靠。能看见什么、能抓哪里、能不能推、会不会滑，这些才是机器人真正需要的视觉问题。
