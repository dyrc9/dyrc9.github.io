Embodied intelligence notes 8 - memory and long-horizon tasks

前面几篇写了 VLA、diffusion policy、evaluation、sim-to-real、data flywheel 和 world model。这里补一个经常被低估的问题：memory。

机器人任务不是每次都只看当前一帧。很多时候，当前图像看不出完整状态，必须记住之前发生过什么。

比如“把桌上的杯子放进柜子”：

  • 杯子刚才在哪里？
  • 现在是不是已经抓住了？
  • 柜门有没有打开？
  • 哪个物体已经处理过？
  • 上一次抓取失败是因为滑落，还是因为没对准？

这些都不是当前一张图一定能回答的问题。长程任务里，memory 不是装饰，而是状态估计的一部分。

partial observation

具身智能里很少真的能看到完整状态。

相机可能被遮挡，物体可能在抽屉里，夹爪背后发生的接触也不一定能看清。形式上更像 POMDP：

那就隐含假设当前观测足够代表状态。短任务里有时勉强可以，长任务里通常不够。

所以 memory 的第一层作用是 belief state：用历史信息估计当前世界到底是什么样。

task progress

长程任务里，策略需要知道自己做到哪一步了。

同样看到一个杯子和一个柜子，动作可能完全不同：

  • 如果还没抓杯子，下一步是抓
  • 如果已经抓住杯子，下一步是移动
  • 如果柜门没开，下一步是先开门
  • 如果杯子已经放好，下一步是确认或结束

这就是 task progress。它不一定能从当前图像直接看出来，尤其是任务中间状态很相似时。

模型不一定显式维护状态机，但它需要有某种等价的进度表示。否则很容易重复做同一件事，比如反复抓已经抓住的物体，或者放完后又继续移动。

episodic memory

一种比较直观的 memory 是 episodic memory：记录这一轮任务中发生过的事件。

可能记录：

  • seen objects
  • object locations
  • completed subgoals
  • failed attempts
  • human intervention
  • environment changes

比如机器人刚才打开了抽屉，当前画面里抽屉只露出一角。memory 可以告诉它：这个抽屉已经开过，不需要再从头找把手。

episodic memory 的价值在于，它保留任务过程，而不是只保留当前视觉特征。

semantic memory

另一类是 semantic memory，更像长期知识。

比如：

  • 杯子通常可以从侧面抓
  • 抽屉需要先拉把手
  • 透明物体视觉识别更容易失败
  • 某个夹爪不适合抓太薄的东西

这些不是某一次 episode 的临时状态，而是跨任务复用的知识。

robot foundation model 里，这部分可能来自大规模数据预训练；真实部署后，也可能来自长期日志总结。

一个实际系统可能同时有两种 memory：

memory representation

memory 可以有很多形式，不一定都是自然语言。

常见表示：

  • recurrent hidden state
  • transformer context
  • object map
  • key-value memory
  • scene graph
  • task state
  • text summary

不同表示适合不同问题。

视觉导航可能需要地图；桌面操作可能需要 object-centric memory；语言交互任务可能需要 text summary；低层控制可能只需要短时间 hidden state。

我现在比较倾向的理解是：不要先问“用不用长上下文”，而要先问“哪些信息需要被保留，保留多久，以什么形式被动作模块使用”。

memory and planning

memory 和 planning 是连在一起的。

planner 需要知道已经完成了哪些子目标，哪些尝试失败过，哪些路径不应该再试。

比如抓取失败后：

  1. 记录失败类型：slip
  2. 更新物体位置
  3. 换抓取方向
  4. 如果连续失败，切换策略或请求帮助

没有 memory，策略可能每次都像第一次尝试一样，重复同一个失败动作。

这也和 evaluation 那篇接上：failure mode 不是只写在报告里，它应该进入下一步决策。

memory can be wrong

memory 不是越多越好，也不是一定可靠。

常见问题：

  • 旧信息过期
  • 视觉误检被记住
  • 任务进度判断错
  • 摘要丢掉关键细节
  • 长上下文里相关信息找不到

比如机器人记住“杯子在桌子左侧”，但中途杯子被碰到了右侧。如果 memory 不更新，反而会误导策略。

所以 memory 需要更新和校验。一个健康的系统应该允许新观测纠正旧记忆，而不是把历史当成绝对事实。

小结

长程具身任务里的 memory 可以先记：

  • 当前观测往往不足以代表完整状态
  • memory 的核心作用是估计状态和记录任务进度
  • episodic memory 记录当前 episode 发生过什么
  • semantic memory 保存跨任务可复用知识
  • memory 表示可以是 hidden state、object map、scene graph 或文字摘要
  • planning 需要记住失败历史，避免重复试错
  • memory 也会错，需要被新观测校验和更新

如果 world model 是“先想一下会发生什么”，memory 更像“别忘了刚才已经发生了什么”。长程机器人任务里，这两件事缺一块都会很难稳。
