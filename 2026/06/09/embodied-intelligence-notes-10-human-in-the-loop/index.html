Embodied intelligence notes 10 - human-in-the-loop and intervention learning

前面写 reward modeling、imitation learning、evaluation 和 data flywheel 时都提到过 human feedback。这里把它单独拎出来。

机器人学习里，人类不只是“先给一批示范数据”的角色。真实系统跑起来之后，人类会在很多地方出现：

  • 任务开始前给指令
  • 演示一个动作
  • 在策略快失败时接管
  • 标注失败原因
  • 判断动作是否安全
  • 给偏好或排序
  • 在不确定时回答问题

这些都可以叫 human-in-the-loop。它的核心不是让人一直遥控机器人，而是把人的判断放到最有信息量的位置。

demonstration is only the start

最常见的人类数据是 demonstration。

比如 teleoperation：

这些数据可以直接做 behavior cloning。问题是 demonstration 通常覆盖的是“人会怎么成功完成任务”，但不一定覆盖机器人自己会怎么失败。

一个 BC policy 部署后，最缺的往往不是标准成功轨迹，而是偏离轨迹后的恢复数据。

所以 demonstration 是起点，不是整个闭环。

intervention

intervention 指的是策略执行过程中，人类发现它快出问题了，于是接管或纠正。

比如：

  • 夹爪快撞到桌面，人类暂停
  • 抓取点偏了，人类修正末端位姿
  • 物体滑落，人类重新摆正
  • 策略重复尝试同一个失败动作，人类切换子目标

这些介入很有信息量，因为它们发生在 policy 不会处理的状态附近。

可以把 intervention 看成一种“自动发现难点”的数据收集方式。策略哪里不稳，人类就在哪里出现；这些地方正是下一轮训练最该补的地方。

learning from intervention

intervention 数据可以有几种用法。

第一种是直接把人类接管后的动作加入数据集：

这很像 DAgger：让 policy 先跑，收集它自己访问到的状态，再让专家告诉它应该怎么做。

第二种是把 intervention 当成负反馈。比如某个状态动作对触发了人类接管，说明这个动作可能不安全或不可靠。

第三种是学习 intervention predictor：

如果模型预测某个动作大概率需要人类介入，就可以提前降速、换策略，或者请求确认。

preference feedback

有些任务很难写 reward，但人类可以比较两个结果。

比如两个整理桌面的轨迹：

  • A 更快，但动作有点危险
  • B 慢一点，但更稳

人类可以选择更偏好的轨迹。然后训练 reward model 或 preference model：

这和 RLHF / preference learning 的结构很像，只是机器人里反馈成本更高，而且安全边界更硬。

偏好反馈适合处理那些不容易写成标量 reward 的东西：

  • 动作是否自然
  • 是否太接近碰撞
  • 路径是否稳
  • 是否需要太多人工重置
  • 是否符合任务习惯

但它也有噪声。不同人对“稳”和“自然”的判断可能不一样，所以协议要尽量清楚。

active query

人类反馈贵，所以不应该每一步都问。

更合理的是 active query：只在模型不确定或风险较高时问人。

比如：

  • 目标物体有歧义：问用户要哪个杯子
  • affordance 不确定：问这个把手能不能拉
  • 安全风险高：请求确认
  • 连续失败多次：请求人类演示一次

labeling failure

失败数据如果只写 failed，价值有限。

人类可以标注失败类型：

  • perception error
  • wrong object
  • bad grasp point
  • collision
  • slip
  • timeout
  • unsafe motion
  • task misunderstanding

这些标签可以直接指导下一步修什么。

如果主要是 perception error，就补视觉数据；如果是 slip，就补抓取和触觉；如果是 task misunderstanding，就检查语言 grounding 或 planner。

这也是为什么 evaluation protocol 不只是测试报告。它应该进入数据闭环。

human workload

human-in-the-loop 的风险是把系统做成“表面自动，实际全靠人盯着”。

所以需要记录人类成本：

  • intervention count
  • intervention duration
  • labeling time
  • reset time
  • number of active queries
  • false alarms

如果一个策略成功率很高，但每次都要人确认三次，那部署价值仍然有限。

真正希望看到的是：随着数据闭环推进，human workload 下降，而不是只是成功率上升。

safety boundary

在人类介入系统里，安全边界要很清楚。

有些动作不能等人来救：

  • 高速撞击
  • 超出 workspace
  • 接近人类身体
  • 夹爪施力过大
  • 进入不可恢复姿态

这些应该由 safety layer 或 controller 先拦住，而不是只靠人类监督。

human feedback 适合改进策略，但不能替代底层安全约束。

小结

human-in-the-loop 这条线可以先记：

  • demonstration 只是起点，部署后的 intervention 更接近真实失败分布
  • intervention 数据可以做纠错示范、负反馈或 intervention prediction
  • preference feedback 适合表达难写成 reward 的质量判断
  • active query 可以把人类反馈用在最有信息量的位置
  • failure labeling 能把评估结果变成下一轮数据收集计划
  • human workload 本身也应该被评估
  • 安全边界不能只靠人类接管

对真实机器人来说，人类不是训练前的一次性数据源，而是系统改进闭环的一部分。好的设计应该让人类越来越少地救场，但每次出现时都能留下有用的数据。
