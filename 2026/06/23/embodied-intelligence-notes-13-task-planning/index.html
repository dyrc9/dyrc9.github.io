Embodied intelligence notes 13 - task planning and skill composition

比如一句指令：

它不是一个单步动作。机器人需要找到杯子、确认抽屉、打开抽屉、抓杯子、移动、放进去、检查结果。中间任何一步失败，都要恢复或重新规划。

subgoals

subgoal 是任务拆解的中间状态。

比如“把杯子放进抽屉”可以拆成：

  1. locate cup
  2. locate drawer
  3. open drawer
  4. grasp cup
  5. move cup to drawer
  6. release cup
  7. verify cup inside drawer

这些 subgoal 不一定要人工写死。VLA 或 planner 可以动态生成。但无论怎么生成，好的 subgoal 应该有两个特点：

  • 可执行
  • 可验证

如果一个 subgoal 没法判断是否完成，后面的 planning 就会变得很脆。

skills

skill 是比低层动作更大的控制单元。

常见 skill：

  • reach
  • grasp
  • lift
  • place
  • push
  • pull
  • open
  • insert
  • recover

skill 可以看成 option，也可以看成一个带参数的 policy：

比如 grasp(cup, side) 或 open(drawer, handle)。

这样做的好处是，高层 planner 不需要每一步都处理关节控制。它只要选择 skill 和参数，让低层 controller 或 policy 执行。

precondition and effect

如果想让 skill 能组合，就要知道它什么时候能用、执行后会改变什么。

这就是 precondition 和 effect。

比如 grasp(cup)：

  • precondition：杯子可见、可达、夹爪空
  • effect：杯子在夹爪里，或者 grasp failed

place(cup, drawer)：

  • precondition：杯子已被抓住、抽屉打开、目标区域可达
  • effect：杯子在抽屉里，夹爪空

这些描述不一定要完全符号化，但系统里需要有等价的判断。否则 planner 可能在抽屉没打开时尝试放杯子，或者在没抓住杯子时尝试移动杯子。

verification

每个 skill 结束后都应该检查结果。

这比“按计划执行到底”更重要。真实机器人里，skill 可能失败：

  • grasp 没抓住
  • pull 没拉开
  • place 后物体滑出
  • 插入时卡住
  • 视觉目标丢失

所以长程执行应该是闭环：

verification 可以来自视觉、触觉、force signal、state estimator 或 human feedback。

如果没有 verification，planner 只是在假设世界按计划变化。

recovery loop

长程任务一定要考虑 recovery。

失败不是只返回 failed，而是要问：

  • 能不能重试同一个 skill？
  • 要不要换参数？
  • 要不要回到上一个 subgoal？
  • 要不要重新观察？
  • 要不要请求人类介入？
  • 要不要安全退出？

比如抓取失败：

  1. 重新检测杯子位置
  2. 换一个 grasp point
  3. 降低速度
  4. 如果连续失败，询问或切换策略

这就把 memory、evaluation、human-in-the-loop 和 runtime safety 接起来了。

language planner

现在很多系统会用语言模型做高层 planner。它的优势是能处理自然语言任务和常识拆解。

但直接让语言模型决定机器人动作有几个问题：

  • 它可能生成不可执行步骤
  • 它不知道当前机器人能力边界
  • 它可能忽略实时观测
  • 它不一定能判断执行是否成功

所以更稳的结构是让语言模型做候选计划，而不是最终执行者。

高层语言能力有价值，但必须接到机器人自己的 perception、skill library 和 safety layer 上。

小结

task planning and skill composition 可以先记：

  • 长程任务不适合只靠一次 policy call
  • subgoal 应该可执行、可验证
  • skill 是可复用的动作单元，可以带参数
  • precondition 和 effect 决定 skill 能不能组合
  • 每个 skill 后都需要 verification
  • recovery loop 是长程任务稳定性的关键
  • LLM 更适合做候选 planner，不应绕过机器人系统检查
  • world model 可以帮助评估 skill 后果

具身智能里的长程能力不是一个大模型单独“想明白”就够了。更像是 planning、skills、verification、memory、safety 和 recovery 这些部件一起把任务稳稳接住。
