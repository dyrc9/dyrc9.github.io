Embodied intelligence notes 2 - VLA and robot data

上一篇把具身智能先从RL角度接了一下：观测、动作、reward、sim-to-real。这一篇换一个更工程一点的视角，看现在常见的 Vision-Language-Action model。

VLA 这个词很直观：输入视觉和语言，输出动作。形式上可以写成：

at ∼ π(at|ot, l)

其中ot是图像或多模态观测，l是语言指令，at是机器人动作。

看起来只是多模态模型多接了一个action head，但实际难点大多不在“多模态”这三个字上。

语言指令不是任务本身

语言指令一般是高层的，比如：

  • put the cup on the plate
  • open the drawer
  • pick up the red block

但机器人执行的是低层连续动作。中间缺了很多东西：

  • 当前场景里杯子在哪里
  • 杯子能不能抓
  • 应该从哪个方向接近
  • 抓起来之后路径怎么走
  • 如果没抓稳要不要重试

所以语言更像给了一个goal，而不是完整policy。模型需要把goal落到一串可执行动作上。

这也是为什么长程任务麻烦。语言里一句话，可能对应几十到几百步控制。

action tokenization

如果用Transformer做策略，一个自然想法是把动作也离散化成token。比如把末端位姿增量分桶：

at = (Δx, Δy, Δz, Δr, Δp, Δy, g)

每个维度离散成若干bin，再像语言模型一样预测下一个action token。

好处是训练形式统一，可以直接做behavior cloning。坏处也很明显：

  • 离散太粗，动作不精细
  • 离散太细，action space太大
  • 不同机器人动作定义不同，很难共享
  • 连续控制里的平滑性会被破坏

另一种做法是输出连续动作分布，比如高斯分布或扩散模型。扩散策略适合表达多峰动作：同一个任务可能左抓也行，右抓也行，直接回归平均值反而会学出一个尴尬动作。

behavior cloning still strong

很多VLA训练本质上还是大规模behavior cloning：

maxθ∑(o, l, a) ∈ Dlog πθ(a|o, l)

这和前面Offline RL里的BC一样，只是输入更复杂、数据更多。

BC强的前提是数据覆盖好。机器人数据一旦覆盖不好，就会出现 compounding error：策略稍微偏离专家轨迹，就进入数据里没见过的状态，然后越来越不知道怎么回。

所以数据质量比模型名字更重要。一个小而干净的数据集，有时比一个混乱的大数据集更有用。尤其是机器人数据里，不同操作者、不同控制频率、不同相机标定都会带来分布差异。

data mixture

具身数据通常不是一种数据，而是混在一起：

  • human teleoperation
  • scripted policy
  • simulation rollout
  • robot self-collected data
  • video-only data

video-only data没有动作标签，但能提供视觉常识和任务阶段信息；teleoperation有动作，但规模小；simulation便宜，但有sim-to-real gap。

一个实际问题是：这些数据应该怎么混？如果全部丢进一个模型，模型可能学到冲突的动作风格。比如同一个任务，有人先旋转物体再抓，有人直接抓；同一条语言指令，在不同机器人上动作空间也不同。

这时需要显式告诉模型 embodiment 信息，或者把动作映射到更统一的空间，比如 end-effector pose / skill / waypoint。

hierarchy

长程任务通常需要分层：

  • high-level planner：把任务拆成子目标
  • mid-level policy：选择skill或waypoint
  • low-level controller：执行连续控制

比如“收拾桌面”可以拆成：

  1. 找到杯子
  2. 抓杯子
  3. 移动到水槽
  4. 放下

如果让一个端到端模型直接从图像和语言输出电机命令，它也许能在短任务上工作，但长任务里很容易丢状态。分层的好处是每层解决的问题更清楚。

这和RL里的option / skill learning 有点像。动作不一定是最底层控制量，也可以是一个可复用的skill。

evaluation

具身智能的评估也很烦。NLP里可以跑benchmark，机器人里一次实验可能要人工重置环境。

常见指标：

  • success rate
  • average return
  • task completion time
  • number of interventions
  • robustness under visual/domain shift

但success rate本身也可能不够。一个策略成功率高，但动作很危险、路径很奇怪，真实部署也不能用。机器人策略需要看稳定性、安全性和可恢复性。

和RL的关系

VLA不是把RL替代掉，而是把RL的位置改了。

BC/VLA可以给一个不错的初始化；RL可以在下面这些地方继续发挥作用：

  • 用少量真实反馈微调
  • 对稀疏成功信号做优化
  • 学习可复用skill
  • 为planner提供value function
  • 在仿真里做策略改进

也就是说，RL从“从零学会任务”变成“在已有行为先验上改进任务”。

小结

VLA这条线先按几个问题记：

  • 语言只是goal，不是完整控制程序
  • action representation 决定策略到底能不能精细执行
  • 大规模BC仍然很强，但怕distribution shift
  • robot data mixture 比普通视觉/语言数据更难处理
  • 长程任务需要分层或记忆

后面如果继续看具体论文，可以先问它解决的是哪一层：视觉表征、语言 grounding、动作建模、数据混合、还是长期规划。
