Embodied intelligence notes 3 - diffusion policy and action chunking

上一篇 VLA 里写到 action representation。这里继续沿着动作建模往下看：为什么机器人策略里会出现 diffusion policy 和 action chunking。

先说一个直觉：机器人动作不是分类题。一个状态下“正确动作”可能不止一个，而且动作之间有连续性。抓杯子可以从左边抓，也可以从右边抓；先靠近再闭合夹爪，中间动作还要平滑。

如果直接做回归，模型会倾向于预测平均动作。两个可行动作一平均，可能就变成不可行动作。

multi-modal action

假设同一个观测下有两条专家轨迹：

  • 从左边绕过去抓
  • 从右边绕过去抓

如果用 MSE 训练一个确定性 policy，它可能学到中间路线。中间路线不一定安全，也不一定能抓到。

这就是 multi-modal action distribution 的问题。动作分布不是单峰高斯，而是可能有多个峰。

BC本身没有错，问题是policy family太弱。只输出一个均值时，模型没法表达“这几个动作都可以，但不要取平均”。

diffusion policy

Diffusion policy 的想法是把动作序列当成生成对象。训练时给真实动作加噪声，模型学习一步步去噪。

粗略过程：

  1. 取一段专家动作序列
  2. 加噪声得到 noisy action
  3. 模型根据观测和噪声动作预测噪声或干净动作
  4. 推理时从噪声开始迭代去噪，生成一段动作

它的好处是能表示复杂动作分布。多个可行动作模式不需要被压成一个平均值，而可以通过采样得到不同动作序列。

缺点也有：推理要多步去噪，比一次前向更慢；另外生成出来的动作还要足够平滑、可执行。

action chunking

Action chunking 是另一个很常见的动作表示方法：不是一次只预测下一步动作，而是预测未来一小段动作。

执行时可以只执行前几步，然后重新观测、重新预测。这有点像 MPC 的 receding horizon。

为什么要chunk？

  • 单步动作太短，看不到局部意图
  • 多步动作更容易保持平滑
  • 对遥操作数据更友好，因为人类动作天然有连续片段
  • 可以减少高层模型调用频率

但chunk太长也不好。环境变化后，后面的动作可能已经不适合了。所以实际会有一个折中：预测一段，但只执行其中一小段。

temporal ensembling

如果连续多次预测action chunk，同一个未来时刻可能会被预测多次。可以把这些预测做加权平均，让动作更平滑。

比如第t+3步动作，可能来自：

  • 当前时刻预测的第3个动作
  • 上一时刻预测的第4个动作
  • 再上一时刻预测的第5个动作

这些预测融合起来，能减少抖动。

这个技巧很工程，但在机器人里很重要。动作抖动不是视觉上不好看，而是可能真的损伤执行或导致失败。

relation to VLA

VLA常常强调视觉和语言，但真正落到机器人上，还是要把动作输出做好。

可能的输出形式有几类：

  • discrete action token
  • continuous action regression
  • action chunk
  • diffusion-generated action sequence
  • high-level skill / waypoint

这些不是互斥的。一个系统可以高层用语言模型规划skill，低层用diffusion policy生成连续动作。

和RL的关系

Diffusion policy大多还是 imitation learning 思路。它解决的是“如何拟合专家动作分布”。

RL可以接在后面做几件事：

  • 用真实成功信号微调动作生成模型
  • 对action chunk做value-guided selection
  • 在多个采样动作里选价值更高的
  • 学一个critic判断生成动作是否安全/有效

这里有点像把policy当作proposal distribution，RL或planner负责选择和改进。

小结

这篇先记几个点：

  • 机器人动作常常是multi-modal，直接MSE回归会平均掉动作模式
  • diffusion policy能表达复杂动作分布，但推理更重
  • action chunking让策略输出一段动作，提升局部一致性
  • temporal ensembling能减小动作抖动
  • VLA落地时，action head 的设计和视觉语言部分一样关键

越往具身智能走，越会发现“动作”不是模型最后接一个线性层那么简单。动作表示本身就是问题的一大半。
